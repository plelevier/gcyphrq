## Summary

Add `random()` scalar function and fix pre-existing bug with floating-point literals in WHERE comparisons.

## Changes

### `random()` function

New scalar function returning a random float between 0 and 1. Works in all expression contexts (RETURN, WHERE, WITH, ORDER BY, CASE, list comprehensions).

**Examples:**
```cypher
RETURN random() AS r                              -- random float 0..1
MATCH (n) RETURN n.name ORDER BY random()          -- shuffle results
MATCH (n) RETURN n.name LIMIT 3 ORDER BY random()  -- random sample
MATCH (n) WHERE random() < 0.5 RETURN n.name       -- probabilistic filter
RETURN [x IN [1,2,3] | random()] AS values         -- in list comprehension
```

**Files:**
- `src/engine/expression.ts` — Added `random` case in `evaluateStringFunction`
- `test/random-function.test.ts` — 22 tests (basic, shuffling, sampling, arithmetic, CASE, WHERE, list comprehension, edge cases, statistical distribution)

### Bug fix: `DoubleLiteral` in WHERE comparisons

The ANTLR4 grammar uses `DoubleLiteral` for floating-point numbers, but the parser only handled `FloatLiteral`. This caused **any float literal on the right side of a WHERE comparison** to be extracted as `undefined`.

**Affected queries (now fixed):**
```cypher
MATCH (n) WHERE n.score > 0.5 RETURN n.name       -- was: undefined WHERE
MATCH (n) WHERE random() < 0.5 RETURN n.name      -- was: undefined WHERE
MATCH (n) WHERE timestamp() > 0.5 RETURN n.name   -- was: undefined WHERE
```

**Files:**
- `src/engine/tree-utils.ts` — Added `DoubleLiteral` to Ctx mapping
- `src/engine/expression-parser.ts` — Added `DoubleLiteral` extraction in `extractLiteral`
